Professioneel Vakmanshap

Ik vind professioneel vakmanschap een van de belangrijkste competentie. Vooral bij mijn product van het tweede project, waar Agile Development werd gebruikt, vond ik het belangrijk dat kwaliteit geleverd moest worden. Voor het product hadden wij een 8,4 gehaald en voor de techno (waar individueel de code wordt besproken) een 8,6. Kwaliteit laat zien dat de code schoon is en dat de UI er goed uit zien.

Onderzoekend Vermogen

Zonder onderzoek kan je geen product maken, of laat staan een project starten. Op het HvA in mijn tweede semester heb ik geleerd hoe ik een goed moest researchen. Dit heeft natuurlijk goed geholpen en heb ik ook een redelijk cijfer gehaald. Het onderzoeken moet toch wel wat beter bij mij, vooral field researching. Hiervoor moet ik ook goede, niet-beantwoorde vragen bedenken, wat ook niet makkelijk is voor mij.

Leervermogen

Ik heb veel nare dingen meegemaakt tijdens projecten. Ik heb geleerd dat je niet kan werken in een teamverband als iemand zijn fouten niet toegeeft of als hij/zij te vaak afwezig is. Ik moet wel deze mensen bedanken, want zij hebben mij laten zien hoe je NIET moet gedragen. Ik kan namelijk kritiek goed accepteren en dat liet ik zien in de PAD project.

Communicatief Vermogen

Dit is zeker de belangrijkste competentie die er is in de ICT-wereld. Je kan ook communiceren met de klanten zonder te spreken. Aangezien ik belangrijk vind dat de user interface er goed uit moet zien, is het ook gebruiksvriendelijk voor de gebruiker en communiceert de applicatie goed met de klant. Bij de twee projecten werden mijn designs gebruikt en geprezen. Wat ook een vaardigheid van mij is, is het afstemmen van taalgebruik van type persoon. Bij een product owner is het belangrijk dat je formeel praat, maar bij een project coach kan er wat informeler gepraat worden.
